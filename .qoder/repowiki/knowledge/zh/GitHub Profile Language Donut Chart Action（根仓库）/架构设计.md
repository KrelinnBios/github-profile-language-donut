仓库作为单一 GitHub Action 发布，入口为 `action.yml`，采用 `runs.using: composite` 编排，在 bash shell 中直接调用 `src/generate.py` 完成核心逻辑；输入参数（`github-token`、`config-path`、`readme-path`、`output-directory`、`output-prefix`）与输出（`image`、`changed`）通过 `steps.generate.outputs.*` 透传。CI 测试位于 `.github/workflows/test.yml`，使用 `python -m unittest discover -s tests` 运行单元测试。文档提供简体中文/繁体中文/英文三语 README，配合 `examples/` 下的配置与工作流模板供使用者复制。依赖方向单向：Action → Python 标准库 + GitHub API，无第三方包依赖。
