仓库以单一 GitHub Composite Action 形式发布，入口 `action.yml` 通过 `runs.using: composite` + `shell: bash` 调用 `src/generate.py`，将输入参数（`github-token`、`config-path`、`readme-path`、`output-directory`、`output-prefix`）映射为命令行参数，并通过 `steps.generate.outputs.*` 透传 `image`、`changed` 两个输出。

核心逻辑位于 `src/language_donut/` 包内，按职责分层：
- `config.py`：加载 `language-donut.config.json`，合并 `DEFAULT_CHART` / `DEFAULT_THEME` / `DEFAULT_COLORS` 三组默认值，支持旧键名（`ring_center_x` → `donut_center_x` 等）向后兼容；
- `github.py`：封装 GitHub REST API 调用（`/users/:owner/repos`、`/repos/:owner/:repo/languages`），从环境变量解析 `GITHUB_REPOSITORY`、`GITHUB_TOKEN`、`GITHUB_API_URL`，分页拉取公开仓库并按 `excluded_repositories`、`include_archived`、`include_forks` 过滤；
- `chart.py`：基于配置与语言统计构建 SVG 字符串；
- `colors.py`：颜色映射与主题处理；
- `output.py`：写入版本化 SVG、更新 README 引用、比较变更并设置 Action outputs。

`src/generate.py` 作为 CLI 入口，仅负责参数解析与模块编排，无业务逻辑。依赖方向单向：Action → generate → language_donut.{config,github,chart,output}，仅使用 Python 标准库（`argparse`、`json`、`urllib.request`、`collections.Counter`、`pathlib`），无任何第三方包依赖。CI 由 `.github/workflows/test.yml` 在 `ubuntu-latest` 上通过 `python -m unittest discover -s tests` 运行单元测试。
